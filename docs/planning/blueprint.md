# 로또 번호 추천 MVP — Codex 작업 지시서

## 1) 제품 컨셉

* 목적: **바이럴 실험용** 로또 번호 추천 페이지(MVP)
* 핵심 경험: **랜덤 객관식 3문항(2개 콘크리트 + 1개 상징)** → “지금 이 순간”에 맞는 조합을 받은 느낌
* 입력 프릭션 최소화: **텍스트 입력 없음**, 선택만 3번
* AI/서버 없이 **정적 사이트만으로 구현**

## 3) UX 플로우

### 기본 흐름

1. 첫 화면: 타이틀/서브 + 시작 버튼(또는 바로 Q1 노출)
2. Q1 선택 → Q2 선택 → Q3 선택
3. 결과 화면: 번호 애니메이션 출력 + 복사/이미지 저장 + 다시하기

### 질문 표시 방식

* 한 화면에 질문 1개(몰입감)
* 상단 진행표시: `1/3`, `2/3`, `3/3`
* 선택하면 자동으로 다음 질문으로 이동

### 결과 화면 구성

* 상단 고정 문구(위 ‘통일 문구’)
* 추천 번호 6개(정렬, 중복 없음)
* 보조 번호 2개(옵션)
* **추천 퍼센트(바이럴 장치)**

  * 예: `오늘 이 조합을 받은 사람 1.7%`
  * 범위: 1.0% ~ 3.0% (소수점 1자리)
* 버튼

  * `📋 번호 복사`
  * `🖼 이미지로 저장`(가능하면)
  * `🔄 다시하기`

### 재시도 정책

* `다시하기` 누르면 새 질문 3개를 뽑아 다시 진행
* 같은 날 같은 선택이면 같은 번호가 나오는 “운명감” 옵션 가능(아래 로직 참고)

## 4) 디자인 가이드

### 🎨 전체 컨셉 방향

**파스텔 기반, 캐릭터 감성, 부드러운 곡선 중심 레이아웃**을 활용한다.
단, 과하게 유아틱하지 않도록 톤은 정돈된 형태로 유지한다.

키워드:

* 파스텔
* 둥근 형태
* 캐릭터/마스코트 느낌
* 말랑한 UI
* 가벼운 축제 분위기

---

### 🎨 색감 방향

* 메인 배경: 부드러운 크림톤 또는 연핑크/연보라 계열
* 카드 배경: 화이트 또는 아주 연한 색조
* 포인트 컬러: 노랑, 민트, 코랄, 하늘색 등 파스텔 계열
* 숫자는 또렷하게 보이도록 짙은 네이비/차콜 사용

색감은 대비가 강하지 않고, 전반적으로 "말랑하고 귀여운" 분위기 유지.

---

### 🧸 그래픽 요소

* 간단한 일러스트(별, 구름, 동그라미, 작은 캐릭터 등) 소량 사용
* 배경에 작은 장식 요소 배치 가능
* 숫자 등장 시 반짝이는 작은 이펙트 추가 가능
* 과한 애니메이션, 화려한 그래디언트 금지

목표는 "운세 보러 온 느낌" + "가벼운 놀이"의 중간 지점.

---

### 🔢 번호 표현 방식

* 6개 숫자는 동그란 칩(원형 배지) 형태로 표시
* 각 숫자 배경은 랜덤 파스텔 색상 적용
* 숫자는 크고 굵게
* 보조 번호는 조금 더 작은 원형 칩

예시 스타일:
( ● 8 ) ( ● 17 ) ( ● 23 ) ( ● 29 ) ( ● 36 ) ( ● 44 )

---

### 🪄 결과 영역 분위기

결과 카드 상단에 작은 반짝이 이모지 또는 별 아이콘 추가

고정 문구:
“지금 당신의 상태에 딱 맞는 로또 조합을 찾았어요!”

문구 아래에 퍼센트 배치:
“오늘 이 조합을 받은 사람 1.7%”

전체적으로 장난스럽지만 신뢰감을 해치지 않는 톤 유지.

---

### 🧩 인터랙션

* 질문 선택 시 카드가 살짝 튀어오르는 애니메이션
* 다음 질문으로 넘어갈 때 슬라이드 전환
* 번호 등장 시 0.2~0.3초 간격으로 순차 등장

너무 빠르지 않게, 약간 기대감을 주는 템포.

## 6) 번호 생성 로직(단순/설명 가능한 수준)

### 목표

* 선택 3개로부터 **seed**를 만들어 1~45 숫자 6개(중복 없음)를 생성
* 과하게 복잡하지 않게

### 권장 구현

1. 선택값을 문자열로 합치기

   * 예: `Q3:0|Q11:2|S2:3|YYYYMMDD`
2. SHA-256 해시
3. 해시 바이트를 순회하며 1~45 숫자 뽑기

   * `n = (byte % 45) + 1`
   * set에 넣어 중복 제거
   * 6개 될 때까지 진행
4. 정렬 후 표시
5. 보조 번호 2개(선택): 해시 다음 바이트로 같은 방식

### 추천 퍼센트(바이럴 장치)

* 난수로 1.0~3.0 사이 생성(소수점 1자리)
* 예: `((seedNumber % 21) / 10 + 1.0)` 형태로 seed 기반이면 “그럴듯한 일관성” 확보 가능

## 7) 구현 요구사항(코드)

### 기술 스택

* Vite + React + TypeScript
* 정적 배포(Cloudflare Pages)

### 컴포넌트/상태

* 상태

  * `step: 1|2|3|"result"`
  * `selected: { q1, q2, q3 }`
  * `questions: [Question, Question, Question]` (선정된 3개)
  * `numbers: number[]`
  * `bonus?: number[]`
  * `percent: string` (예: `1.7%`)

### 기능

* 3문항 랜덤 선택 규칙 준수(2 콘크리트 + 1 상징)
* 버튼

  * 복사: `navigator.clipboard.writeText("8 17 23 29 36 44")`
  * 이미지 저장(가능하면): 결과 영역을 캡처하여 다운로드(라이브러리 사용 가능)
* 다시하기: 새 질문 뽑고 step=1로 리셋

## 8) Codex 작업 지시(실행 순서)

1. Vite React TS 프로젝트 생성
2. 질문 풀을 상수 데이터로 구현
3. 3문항 랜덤 선택 로직 구현(규칙 포함)
4. 단계형 UI 구현(1/3, 2/3, 3/3)
5. seed/해시 기반 번호 생성 구현(Web Crypto API)
6. 결과 화면(통일 문구 + 번호 + 퍼센트 + 버튼)
7. Cloudflare Pages 배포 설정

   * build: `npm run build`
   * output: `dist`
8. README에 로컬 실행/배포 방법 정리

## 10) 1차 MVP 범위 결정(추가)

- 결과 화면의 `이미지로 저장` 기능은 1차 MVP에서 제외한다.
- 1차는 `번호 복사`와 `다시하기`에 집중하고, 이미지 저장은 2차 개선 항목으로 분리한다.
